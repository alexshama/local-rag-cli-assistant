# Local RAG Assistant

Локальный RAG-ассистент на Python с двумя интерфейсами:

- CLI для работы из терминала;
- Telegram-бот для общения через Telegram.

Проект индексирует локальные текстовые документы в ChromaDB, ищет по ним релевантные чанки, передаёт контекст в OpenAI и кэширует ответы в SQLite. Он задуман как понятный учебный проект без тяжёлой архитектуры, но с аккуратными путями, логированием и переиндексацией.

## Что умеет проект

- загружает локальные источники знаний из `assistant_api/data/`;
- строит локальный индекс в ChromaDB;
- ищет релевантный контекст по вопросу;
- отправляет контекст в OpenAI для генерации ответа;
- кэширует ответы в SQLite;
- поддерживает фильтрацию по источникам через `@python` и `@docs`;
- пишет логи в консоль и в `app.log`;
- умеет принудительно переиндексировать базу знаний;
- работает как CLI и как Telegram-бот.

## Как это работает

Пайплайн одинаковый и для CLI, и для Telegram:

1. пользователь задаёт вопрос;
2. приложение проверяет SQLite-кэш;
3. если ответа в кэше нет, создаётся embedding запроса;
4. ChromaDB ищет ближайшие чанки по локальной базе знаний;
5. найденный контекст отправляется в OpenAI;
6. модель возвращает финальный ответ;
7. результат сохраняется в кэш.

По умолчанию используются два источника знаний:

- `docs` — общие материалы по AI / ML / RAG;
- `python` — база знаний по Python.

## Установка

1. Клонируйте репозиторий:

```bash
git clone <YOUR_GIT_REMOTE_URL>
cd <YOUR_PROJECT_DIRECTORY>
```

2. Создайте виртуальное окружение:

```bash
python -m venv .venv
```

3. Активируйте его.

Windows PowerShell:

```powershell
.venv\Scripts\Activate.ps1
```

Windows CMD:

```cmd
.venv\Scripts\activate.bat
```

Linux / macOS:

```bash
source .venv/bin/activate
```

4. Установите зависимости:

```bash
pip install -r requirements.txt
```

## Переменные окружения

Создайте `.env` на основе шаблона:

```bash
cp .env.example .env
```

Минимальный набор переменных:

```env
OPENAI_API_KEY=your-openai-api-key-here
TELEGRAM_BOT_TOKEN=your-telegram-bot-token-here
```

Важно:

- не публикуйте `.env` в GitHub;
- не коммитьте реальные ключи;
- если ключ уже светился локально или в истории git, перевыпустите его.

## Запуск CLI

Обычный запуск:

```bash
python assistant_api/app.py
```

С подробными логами:

```bash
python assistant_api/app.py --debug
```

С принудительной переиндексацией:

```bash
python assistant_api/app.py --reindex
```

## Пример использования CLI

Обычный вопрос:

```text
💭 Ваш вопрос: Что такое RAG?
```

Вопрос только по Python:

```text
💭 Ваш вопрос: @python Что такое переменная?
```

Доступные команды в CLI:

- `stats` — статистика индекса и кэша;
- `clear` — очистка SQLite-кэша;
- `reindex` — принудительная переиндексация базы знаний;
- `exit` / `quit` / `q` — выход.

## Запуск Telegram-бота

Обычный запуск:

```bash
python telegram_bot.py
```

С подробными логами:

```bash
python telegram_bot.py --debug
```

С принудительной переиндексацией при старте:

```bash
python telegram_bot.py --reindex
```

## Пример использования Telegram-бота

После запуска бота можно писать ему обычные вопросы:

```text
Что такое embeddings?
```

Или использовать фильтры по источникам:

```text
@python Что такое tuple?
@docs Что такое retrieval?
```

Доступные команды в Telegram:

- `/start` — приветствие и краткая справка;
- `/help` — как пользоваться ботом;
- `/stats` — статистика системы;
- `/logs` — экспорт ваших взаимодействий с ботом в CSV;
- `/reindex` — принудительная переиндексация.

## Логирование

Проект использует стандартный модуль `logging`.

Логи пишутся:

- в консоль;
- в `app.log` в корне проекта.

По умолчанию используется уровень `INFO`.
Для подробной диагностики используйте `--debug`.

Чувствительные значения в технических логах маскируются перед записью.

## CSV-экспорт взаимодействий

Telegram-бот ведёт отдельный структурированный журнал взаимодействий и по команде `/logs`
отправляет CSV-файл, похожий на учебный формат проверки.

Одна строка CSV соответствует одному пользовательскому запросу.

Поля в CSV:

- `id`
- `timestamp`
- `user_id`
- `username`
- `source`
- `query`
- `response`
- `from_cache`
- `response_time_ms`
- `created`

## Где хранятся локальные данные

Проект хранит служебные файлы в стабильных путях относительно корня репозитория:

- `chroma_db/` — локальная векторная база;
- `semantic_rag_cache_v2.db` — SQLite-кэш;
- `telegram_logs.db` — SQLite-журнал взаимодействий Telegram;
- `app.log` — лог-файл.

Это сделано так, чтобы запуск не зависел от текущей директории.

## Ограничения

- нужен действующий `OPENAI_API_KEY`;
- для Telegram-бота нужен `TELEGRAM_BOT_TOKEN`;
- проект рассчитан на локальные текстовые файлы, а не на большие production-датасеты;
- качество ответа зависит от содержимого `docs.txt` и `python.txt`;
- evaluation через `ragas` не заменяет полноценную ручную проверку;
- нет веб-интерфейса;
- обновление индекса не происходит автоматически при изменении файлов, для этого нужно использовать `reindex`.
